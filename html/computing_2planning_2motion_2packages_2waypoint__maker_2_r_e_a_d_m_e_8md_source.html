64 ### waypoint_loader
65 
66 1. Overview
67 
68  * waypoint_loader
69  - Convert waypoints.csv to ROS message type.
70  - Correspond to the above 3 types of csv.
71  * waypoint_replanner
72  - Adjust waypoints offline (resample and replan velocity)
73 
74 1. How to use
75 
76  * How to start
77  - At `Computing`->`waypoint_loader`:
78  - Check app->`load csv`.
79  If you want to load csv_files, switch to true.
80  Otherwise switch to false.
81  - Check `waypoint_loader` and start.
82  * Idea of velocity replanning
83  - The velocity plan is based on the following idea.
84  - On a straight line, it accelerates to the maximum speed.
85  - At one curve:
86  * Finish decelerating before entering the curve.
87  If the curve is sharper, the more deceleration is required.
88  * Maintain a constant speed in the curve.
89  * Start to accelerate after getting out of the curve.
90  * Detail of app tab
91  - On `multi_lane`, please select multiple input files. If you want lane change with `lane_select`, prepare ver3 type.
92  - Check `replanning_mode` if you want to replan velocity.
93  - On replanning mode:
94  * Check `realtime_tuning_mode` if you want to tune waypoint.
95  * Check `resample_mode` if you want to resample waypoints.
96  On resample mode, please set `resample_interval`.
97  * Velocity replanning parameter
98  - Check `replan curve mode` if you want to decelerate on curve.
99  - Check `overwrite vmax mode` if you want to overwrite velocity of all waypoint.
100  - Check `replan endpoint mode` if you want to decelerate on endpoint.
101  - `Vmax` is max velocity.
102  - `Rth` is radius threshold for extracting curve in waypoints.
103  Increasing this, you can extract curves more sensitively.
104  - `Rmin` and `Vmin` are pairs of values used for velocity replanning.
105  Designed velocity plan that minimizes velocity in the assumed sharpest curve.
106  In the _i_-th curve, the minimum radius _r<sub>i</sub>_ and the velocity _v<sub>i</sub>_ are expressed by the following expressions.
107  _v<sub>i</sub>_ = _Vmax_ - _(Vmax - Vmin)/(Rth - Rmin)_ * _(Rth - r<sub>i</sub>)_
108  - `Accel limit` is acceleration value for limitting velocity.
109  - `Decel limit` is deceleration value for limitting velocity.
110  - `Velocity Offset` is offset amount preceding the velocity plan.
111  - `Braking Distance` is the number of minimum velocity before end point offset.
112  - `End Point Offset` is the number of 0 velocity points at the end of waypoints.

135 ### waypoint_saver
136 
137 1. Overview
138 
139  * waypoint_saver
140  - When activated, subscribe `/current_pose`, `/current_velocity`(option) and save waypoint in the file at specified intervals.
141  * waypoint_extractor
142  - When activated, subscribe autoware_msgs/LaneArray and save waypoint in the file. Input topic name is selectable.
143  * common
144  - `change_flag` is basically stored as 0 (straight ahead),
145  so if you want to change the lane, edit by yourself. (1 turn right, 2 turn left)
146  - This node corresponds to preservation of ver3 format.
147 
148 1. How to use
149 
150  On app:
151  * common
152  - Ref on the `Save File` and specify the save file name.
153  - Select the function you want to use from the `Input Type`.
154  * if `Input Type` == VehicleFootprint (`waypoint_saver`)
155  - Check `Save/current_velocity` if you want to save velocity.
156  In otherwise, saved as 0 velocity.
157  - Using `Interval`, it is set how many meters to store waypoint.
158  * if `Input Type` == LaneArrayTopic (`waypoint_extractor`)
159  - Set lane_array topic name you want to save.
160  - Cache starts at the same time as the node starts up.
161  - The cache is always overwritten while the node is running.
162  - The cache is flushed when the node is closed.

1 # The `object_map` Package
2 
3 This package contains nodes to extract information from the LiDAR sensor and from ADAS Maps.
4 The processed result is published as GridMaps (grid_map) and/or occupancy grids(nav_msgs).
5 
6 ---
7 
8 ## Nodes in the Package
9 
10 ### points2costmap
11 
12 This node reads PointCloud data and creates an occupancy grid based on the data contained in it.
13 
14 The subscribed PointCloud topic should be a processed PointCloud with the ground previously removed.
15 The default input is set as the output PointCloud result from the node `euclidean_clustering` of the `lidar_tracker` package. However, another option might be the output of the `ray_ground_filter` or `ring_ground_filter`, both belonging to the `points_preprocessor` in the *Sensing* package.
16 
17 The values contained within the OccupancyGrid range from 0-100, representing a probability percentage where 0 represents free space and 100 represents occupied space.
18 
19 #### Input topics
20 Default: `/points_lanes` (PointCloud) from euclidean cluster. It contains filtered points with the ground removed. Another possible option is `/points_no_ground`.
21 
22 #### Output topics
23 `/realtime_cost_map` (nav_msgs::OccupancyGrid) is the output OccupancyGrid, with values ranging from 0-100.
24 
25 ##### How to launch
26 It can be launched as follows:
27  1. Using the Runtime Manager by clicking the `points2costmap` checkbox under the *Semantics* section in the Computing tab.
28  2. From a sourced terminal executing: `roslaunch object_map points2costmap.launch`.
29 
30 ##### Parameters available in roslaunch and rosrun
31 * `resolution` defines the equivalent value of a cell in the grid in meters. Smaller values result in better accuracy at the expense of memory and computing cost (default value: 0.1).
32 * `cell_width` represents the width in meters of the OccupancyGrid around the origin of the PointCloud (default: 150).
33 * `cell_height` represents the height in meters of the OccupancyGrid around the origin of the PointCloud (default: 150).
34 * `offset_x` indicates if the center of the OccupancyGrid will be shifted by this distance, to the left(-) or right(+) (default: 25).
35 * `offset_y` indicates if the center of the OccupancyGrid will be shifted by this distance, to the back(-) or front(+) (default: 0).
36 * `offset_z` indicates if the center of the OccupancyGrid will be shifted by this distance, below(-) or above(+) (dDefault: 0).
37 * `points_topic` is the PointCloud topic source.

67 ### wayarea2grid
68 
69 The ADAS Maps used in Autoware may contain the definition of the road areas, which is useful for computing whether a region is drivable or not.
70 This node reads the data from an ADAS Map (VectorMap) and extracts the 3D positions of the road regions. It projects them into an OccupancyGrid and sets the value to `128` if the area is **road** and `255` if it is not. These values are used because this node uses 8-bit bitmaps (grid_map).
71 
72 #### Input topics
73 `/vector_map` (vector_map_msgs::WayArea) from the VectorMap publisher.
74 `/tf` to obtain the transform between the vector map(grid_frame) and the sensor(sensor_frame) .
75 
76 #### Output topics
77 `/grid_map_wayarea` (grid_map::GridMap) is the resulting GridMap with values ranging from 0-255.
78 `/occupancy_wayarea` (nav_msgs::OccupancyGrid) is the resulting OccupancyGrid with values ranging from 0-255.
79 
80 ##### How to launch
81 It can be launched as follows:
82  1. Using the Runtime Manager by clicking the `wayarea2grid` checkbox under the *Semantics* section in the Computing tab.
83  2. From a sourced terminal by executing: `roslaunch object_map wayarea2grid.launch`
84 
85 ##### Parameters available in roslaunch and rosrun
86 * `sensor_frame` defines the coordinate frame of the vehicle origin (default value: velodyne).
87 * `grid_frame` defines the coordinate frame of the map (default: map).
88 * `grid_resolution` defines the equivalent value of a cell in the grid in meters. Smaller values result in better accuracy at the expense of memory and computing cost (default: 0.2).
89 * `grid_length_x` represents the width in meters of the OccupancyGrid around the origin of the PointCloud (default: 25).
90 * `grid_length_y` represents the height in meters of the OccupancyGrid around the origin of the PointCloud (default: 0).
91 * `grid_position_x` indicates if the center of the OccupancyGrid will be shifted by this distance, left(-) or right(+) (default: 0).
92 * `grid_position_y` indicates if the center of the OccupancyGrid will be shifted by this distance, back(-) or front(+) (default: 0).

96 ### grid_map_filter
97 This node can combine sensor, map, and perception data as well as other OccupancyGrids. It generates OccupancyGrids with useful data for navigation purposes. It publishes three different layers. More details are provided in the output topics section below.
98 
99 #### Input topics
100 `/vector_map` (vector_map_msgs::WayArea) from the VectorMap publisher.
101 `/tf` to obtain the transform between the vector map (grid_frame) and the sensor (sensor_frame).
102 `/realtime_cost_map` (nav_msgs::OccupancyGrid) calculated by `points2costmap` in this package.
103 
104 #### Output topics
105 `/filtered_grid_map` (grid_map::GridMap) which contains 3 layers and is also published as in regular OccupancyGrid messages.
106 `distance_transform` (nav_msgs::OccupancyGrid) applies a distance transform to the OccupancyGrid obtained from the `/realtime_cost_map`, allowing us to obtain a gradient of probabilities surrounding the obstacles PointCloud.
107 `dist_wayarea` (nav_msgs::OccupancyGrid) contains a combination of the distance transform and the road data, as described in the `wayarea2grid` node.
108 `circle` (nav_msgs::OccupancyGrid) draws a circle surrounding the `/realtime_cost_map`.
109 
110 The output topics are configured as described in the `grid_map` package, and the configuration file is inside the `config` folder of the package.
111 
112 ##### How to launch
113 It can be launched as follows:
114  1. Using the Runtime Manager by clicking the `grid_map_filter` checkbox under the *Semantics* section in the Computing tab.
115  2. From a sourced terminal by executing: `roslaunch object_map grid_map_filter.launch`.
116 
117 ##### Parameters available in roslaunch and rosrun
118  * `map_frame` defines the coordinate system of the realtime costmap (default value: map).
119  * `map_topic` defines the topic where the realtime costmap is being published (default: /realtime_cost_map).
120  * `dist_transform_distance` defines the maximum distance to calculate the distance transform, in meters (default: 2.0).
121  * `use_wayarea` indicates whether or not to use the road regions to filter the cost map (default: true).
122  * `use_fill_circle` enables or disables the generation of the circle layer (default: true).
123  * `fill_circle_cost_threshold` indicates the minimum cost value threshold value to decide if a circle will be drawn (default: 20)
124  * `circle_radius` defines the radius of the circle, in meters (default: 1.7).

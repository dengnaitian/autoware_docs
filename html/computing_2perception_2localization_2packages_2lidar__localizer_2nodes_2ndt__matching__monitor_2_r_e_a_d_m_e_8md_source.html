1 # ndt_matching_monitor
2 
3 Simple health monitoring and reinitialization for `ndt_matching` localization node of
4  [Autoware](https://github.com/CPFL/Autoware)
5 
6 ## Introduction
7 `ndt_matching` publishes the current robot/vehicle position in the `/ndt_pose` topic and statistics
8 in the `/ndt_stat` topic, such as score, execution time.
9 
10 The score is the result of PCL's function `getFitnessScore()` which measures how well an input point cloud matches
11  the reference pointcloud, in other words the alignment error between the input scan and the reference map.
12  This value can be used to infer the reliability of NDT.
13 
14 `ndt_matching_monitor` subscribes to `/ndt_stat` and `/ndt_pose` topics, and keeps a running average filter on
15 the score value. When the filtered score is beyond some thresholds, `ndt_matching_monitor` will issue
16 the `/initialpose` topic (same as rviz) using the last known "healty" pose to force `/ndt_matching` to initialize.
17 
18 If a GNSS device is available, an automatic reinitialization will be triggered with it. `nmea2pose` node is required.
19 Otherwise, a halt in localization will be started. To reset the halted status, use *Initial Pose* tool in RVIZ.
20 
21 ### How to launch
22 * From a sourced terminal:\
23 `roslaunch lidar_localizer ndt_matching_monitor.launch`
24 
25 * From Runtime manager:\
26 Computing Tab -> lidar_localizer -> ndt_matching_monitor

28 ### Parameters available
29 
30 |Parameter| Type| Description|
31 ----------|-----|--------
32 |`iteration_threshold_warn`|*integer* |Number of maximum iterations before notifying a warning. Default 10.|
33 |`iteration_threshold_stop`|*integer*|Number of maximum iterations before notifying a warning. Default 32. |
34 |`score_delta_threshold`|*double*|Difference between consecutive scores to consider a safe match. Default 14.0|
35 |`min_stable_samples`|*integer*|Minimum number of samples to start monitoring after a reset. Default 30|
36 |`fatal_time_threshold`|*double*|When no GNSS is available a prediction based on the previous samples will be employed. If reinitialization fails the algorithm will stop after n secs. Default 2|

55 ## Contributing
56 `ndt_matching_monitor` is far from complete.
57 For instance, the last known "healthy" pose stated above is just the last `/ndt_pose` value.
58 A Kalman or particle filter might be used to improve this node.

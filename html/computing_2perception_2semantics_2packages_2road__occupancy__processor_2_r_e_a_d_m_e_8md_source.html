1 # Road Occupancy Processor
2 
3 This package generates the occupancy grid indicating the status of the road. It uses the point cloud from a filtered sensor and the ADAS Map data.
4 
5 The occupancy grid can be conceptualized as a one dimensional 8-bit depth bitmap.
6 
7 This package publishes a GridMap/OccupancyGrid with four different possible values:
8 - UNKNOWN
9 - FREE
10 - NO ROAD
11 - OCCUPIED
12 
13 These values can be set using parameters, as described in the *Configuration Parameters* section below.
14 
15 ### Extra requirements for compilation
16 - GridMap (http://wiki.ros.org/grid_map)
17 
18 ### Prerequisites
19 - Localization working (TF between velodyne and world)
20  - PCD Map (map)
21  - TF for the Map (tf map->world)
22  - NDT matching (tf world-> base_link)
23  - Base link to Localizer (tf base_link -> velodyne)
24 - Ground filtering method publishing `/points_ground` and `/points_no_ground`. Autoware has three available:
25  - `ray_ground_filter`
26  - `ring_ground_filter`
27  - `euclidean_cluster` with Planar Ground removal
28 
29 - `wayarea2gridmap` node from Semantics in the Computing Tab
30  - Vector Map in `/vector_map` (also known as ADAS Map)
31  - VectorMap must contain the definition of road areas (way_areas)
32 
33 ### Data subscription
34 This node subscribes to:
35 - Ground filtered in `/points_ground` (sensor_msgs::PointCloud)
36 - Obstacle points in `/points_no_ground` (sensor_msgs::PointCloud)
37 - GridMap containing the road processed in `/grid_map_wayarea` (grid_map::GridMap)
38 
39 ### Data publishing
40 - `gridmap_road_status` publishes the native format of the grid_map package. This can be used to extend the functionalities of this package.
41 - `occupancy_road_status` publishes the native ros `nav_msgs::OccupancyGrid` message.
42 
43 ### How to run
44 From a sourced terminal in the workspace:
45 `roslaunch road_occupancy_processor road_occupancy_processor.launch`
46 
47 ### Quick Setup
48 **This is only a quick guide, each node must be properly configured**
49 
50 1. Load PointCloud Map (Map tab)
51 1. Load VectorMap with wayareas available (Map tab)
52 1. Load TF between map and world frames (Map tab)
53 1. Load TF between base_link and Localizer (Setup tab)
54 1. Launch voxel_grid_filter (Sensing tab/Points Downsampler)
55 1. Launch ray_ground_filter (Sensing tab/Points Preprocessor)
56 1. Launch ndt_matching (Computing tab/Localization)
57 1. Initialize NDT pose
58 1. Launch wayarea2grid (Computing tab/Localization)
59 1. Launch road_occupancy_processor (Computing tab/Localization)

61 ### Configuration parameters
62 * `points_ground_src` (default=`"points_ground"`) defines the PointCloud source topic containing only the ground points.
63 * `points_no_ground_src` (default=`"points_no_ground"`) defines the PointCloud source topic containing only the obstacle points.
64 * `wayarea_topic_src` (default=`grid_map_wayarea`) defines the name of the topic containing the grid_map_msgs:GridMap containing the road areas.
65 * `wayarea_layer_name` (default=`"wayarea"`) defines the name of the layer in the topic `wayarea_topic_src` that contains the road areas.
66 * `output_layer_name` (default=`road_status`) defines name of the output layer in the published GridMap object. If running several instances, each vehicle one can publish a different layer and later add them.
67 * `road_unknown_value` (default=`"128"`) indicates the value to fill in the occupancy grid when a cell is **UNKNOWN**.
68 * `road_free_value` (default=`"75"`) indicates the value to fill in the occupancy grid when a cell is **FREE**. Should be a number between 0-255.
69 * `road_occupied_value` (default=`"0"`) indicates the value to fill in the occupancy grid when a cell is **OCCUPIED**. Should be a number between 0-255.
70 * `no_road_value` (default=`"255"`) indicates the value to fill in the occupancy grid when a cell is **NO ROAD**. Should be a number between 0-255.
71 
72 ### Coordinate Frame
73 The occupancy grid is published in the same coordinate frame as the input GridMap from `/grid_map_wayarea`.
74 
75 ### Color representation using default values
76 
77 **Black** color represents areas not defined as NO ROAD in the vector map.
78 
79 **Dark gray** indicates UNKNOWN areas as detected by this node.
80 
81 **Light gray** means the area is FREE.
82 
83 **White** is used for OCCUPIED areas.
